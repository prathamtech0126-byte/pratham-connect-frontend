Design: States & Error Handling

Section 3 of 3 — Loading, empty, error, and query key strategy

Component States

⏳ Loading
Loading payments...

Spinner centered in the table area. Filter bar remains visible so user can switch period without waiting.

📭 Empty
No payments found for this period.
Total Records: 0

When data.length === 0. Table is hidden; message shown instead.

⚠️ Error
Failed to load payments. Please try again.

Matches the same error card style used in the parent Reports.tsx.

React Query Setup

useQuery("payments-list", filter, startDate?, endDate?)

// queryKey changes → refetch automatically
queryKey: ["payments-list", filter, startDate, endDate]

// staleTime matches Reports page = 2 minutes
staleTime: 1000 * 60 * 2

// only fetch custom when both dates are set
enabled: filter !== 'custom' || !!(startDate && endDate)
